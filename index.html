QuantumPINNs

Physics-Informed Neural Networks for Quantum-Relevant Physical Modeling

We study whether physics-informed neural networks can recover quantum states and quantum dynamics from governing equations and sparse physical structure, without relying on dense supervised solution labels.

The empirical evidence is cumulative: a harmonic-oscillator benchmark establishes near-analytic stationary-state recovery, a time-dependent Schrödinger benchmark demonstrates complex-valued propagation with physics-aware diagnostics, and a shared comparative benchmark tests transferability, ablations, and robustness under a common protocol. The central machine-learning contribution is the comparative evidence, not a single favorable benchmark result.

Empirical Summary

Primary Experimental Results

The paper-level claim is that physics-structured inductive bias makes PINNs more credible for quantum modeling, and that this claim should be defended with both specialist benchmarks and comparative evidence. The three notebooks therefore serve distinct roles: best-case stationary-state fidelity, physics-checked time evolution, and transferability under controlled architectural and robustness studies.

Specialist accuracy

The harmonic-oscillator notebook is the accuracy frontier of the paper. It shows that explicit physical structure can recover the ground state with near-analytic agreement in both wavefunction shape and eigenvalue.

Physics-checked dynamics

The time-dependent Schrödinger notebook demonstrates that the same framework extends beyond static eigenproblems to complex-valued propagation, with density, norm, phase, and transport-aware diagnostics.

Transferability and ablation

The combined benchmark is where the broader machine-learning claim is tested. It identifies which design choices survive a change in problem class and which should be treated honestly as specialist tuning.

Main Figures

Core Empirical Visualizations

These four figures summarize the strongest empirical claims in the submission. They are generated from committed CSV artifacts so the website and README stay synchronized, while the executed notebooks expose the richer supplementary figure sets behind each claim.

Specialist stationary-state accuracy

This benchmark carries the strongest single number in the repository: near-analytic ground-state recovery, supported in the notebook by detailed ground-state, overlap-matrix, and ablation visualizations.

Wavepacket propagation with physics-aware checks

The propagation benchmark begins essentially exact at the initial slice and remains controlled at later times. The executed notebook expands this claim with density heatmaps, phase structure, current snapshots, and Ehrenfest diagnostics.

Architecture sensitivity under a shared protocol

This is where the paper moves from benchmark performance to model-selection evidence. Depth is the strongest lever in the tested sweep, and the 5-layer, 64-unit model is the best shared configuration.

Transferability and robustness

This figure highlights the part of the paper that matters most beyond a single benchmark win: the same formulation remains interpretable across multiple problems and shows only modest degradation under added input noise.

Reproducibility

Reproducibility Protocol

The commands below match the environment assumption already used elsewhere in the repository.

conda activate qaoa
pip install -r requirements.txt

jupyter nbconvert --to notebook --execute --inplace notebooks/pinn_harmonic_oscillator.ipynb
jupyter nbconvert --to notebook --execute --inplace notebooks/pinn_schrodinger.ipynb
jupyter nbconvert --to notebook --execute --inplace notebooks/quantum_pinn_combined.ipynb

jupyter nbconvert --to html notebooks/pinn_harmonic_oscillator.ipynb --output pinn_harmonic_oscillator.html
jupyter nbconvert --to html notebooks/pinn_schrodinger.ipynb --output pinn_schrodinger.html
jupyter nbconvert --to html notebooks/quantum_pinn_combined.ipynb --output quantum_pinn_combined.html
python -m src.train --problem harmonic_oscillator --epochs 5000 --collocation 2000 --model-path model.pt
python -m src.server --model-path model.pt --problem harmonic_oscillator
python -m http.server 8000
